# Clean code VS Dirty code

## Qu'est-ce que du code sale ?

- trop de commentaire pour aucune raison 
- duplication de code 
- mauvais nommage de fonction / variable / fichier 
- aucune convention de nommage 
- long fonction / methode 
- du code spaghetti 
- parametres non utilisés 

## Qu'est-ce que du code propre ?

- code lisible / simple / explicite
- facile a modifier 
- on evite la dette technique 
- bonne structure general du code 
- gestion de packages / modules 
- des commentaires utiles et à jour 
- les principes SOLID 
- programmation defensive ( eviter les effet de bord )
- historique de MaJ claire 
- design paterne
